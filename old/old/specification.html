gx Specification

Execution System

  • Block - Code to execute Actions
    • Ready - ready to run
    • Done - last internal Block has completed
  • Frame - A Block, Input List, Output List and an optional preferred Execution System Actions
    • Schedule - takes a chunk to schedule and an optional preferred execution system
  • Chunk List - Master list of all chunks. Also contains a list of all known Execution Systems.
    Actions
    • Schedule - takes a frame to schedule and an optional preferred execution system
    • Step - sends one frame to an execution system (if one is available to run). If the preferred execution system is the UI execution system, it is guaranteed to run in the UI thread, otherwise it is a request.
    • Idle - determine if there are no Execution Systems with Frames to execute and no Frames are ready to run.
